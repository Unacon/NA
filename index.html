Diferença entre número aleatório e pseudoaleatório

Vamos primeiramente diferenciar números aleatórios dos pseudoaleatórios. Os números pseudoaleatórios são simuladores de aleatoriedade, pois esses números gerados possuem algo tipo de similaridade, se aproximando uns dos outros e tendo algum tipo de correlação calculável.

Outra forma de deduzir um pseudoaleatório, John von Neuman comenta "Qualquer um que considere métodos aritméticos para produzir dígitos está, certamente, cometendo um pecado", de outra forma quer dizer que qualquer programa que tente simular aleatoriedade está criando um pseudoaleatório.

Os números aleatórios não possuem nenhuma forma de cálculo para serem desenvolvidos, os mesmo não podem ser previstos a partir dos seus anteriores. Esses tipos de números são muitos utilizados em criptografia, dinâmica de partículas com dissipação, entre outras.

Tipos de números pseudoaleatório

Gerador Congruente Linear

Esse gerador foi desenvolvido por Derrick Hery Lehmer em 1940, esse gerador possui algumas características citas abaixo:

  1. Considerado rápido

  2. Consome pouca memória

  3. Número possuem correlação

Abaixo temos a função da qual é feita o número pseudoaleatório:

\[ x_{n+1}=(ax_n+c)\%m \begin{cases} m>0\\ 0 \leq a<m\\ 0 \leq s<m\ \Leftrightarrow\ s=x_0\\ c\ \forall\ \mathbb{R} \end{cases} \]

Agora iremos análisar a função do gerador detalhadamente. A variável 'm' serve para delitimar o número sorteado, na qual poderá variar entre '0' a 'm-1'.

A variável 'a' e 'c' serve para modificar na amplitude do sorteio, caso for baixo, os planos de correlação irão se encontrar com mais facilidade, tornando o número encontrado rastreado.

Para tornar o gerador mais rápido, basta diminuir o valor das variáveis, pois conta somente com multiplicações e divisões.

Um ponto negativo desse gerador é caso a variável 'm' seja pequena, o periodo de repetição será curto, podendo gerar conflitos com aleatoridade do gerador.

Abaixo temos um exemplo para \(m=10,\ a=11,\ c=3\ e\ x_0=s=3\):

  • \(x_{n+1}=(ax_n+c)\%m\)

    • Resultado: \[ x_1=6\to x_2=9\to x_3=2\\ x_4=5\to x_5=8\to x_6=1\\ x_7=4\to x_8=7\to x_9=0\\ x_{10}=3\to x_{11}=6\to x_{12}=9 \]

Percebemos que posteriormente da posição '10' os números começam a se repetir, uma forma de amenizar essa situação é na utilização de valores maiores nas variáveis.

Agora iremos demostrar a utilização do Gerador Congruente Linear com valores maiores.

Exemplo para 10000 pontos:

  • \(m=2147483648\)

  • \(a=65539\)

  • \(c=0\)

Código Fonte

Agora iremos entrar na explicação do código fonte, abaixo temos o código completo:

#include <iostream>
#include <opencv2/opencv.hpp>
#include <math.h>
#include <bitset>

using namespace cv;
using namespace std;

int main (int argc, char** argv){
  Mat image,frame,image1,frame1;
  VideoCapture cap;
  int width,height,i,j,teste,rangeinf,rangesup,x=0,y=0,z=0;
  int n,posicao=-1; // Quantidade de Bits.
  char key;

  cap.open(0); // Captura Video

  if(!cap.isOpened()){
    cout << "Erro" << endl;
    return -1;
  }//Verifica se video foi aberto

  //Dimensões do Video
  width=cap.get(CV_CAP_PROP_FRAME_WIDTH);
  height=cap.get(CV_CAP_PROP_FRAME_HEIGHT);

  cout<<"Largura: "<<width<<"\nAltura: "<<height<<endl;
  cout<<"Qual range de trabalho\n";
  cin>>rangeinf;
  cin>>rangesup;

  //Ajusta espaço de aleatoridade
  if(rangeinf>rangesup){
    int k;
    k=rangesup;
    rangesup=rangeinf;
    rangeinf=k;
  }
  //Erro do espaço
  if(rangeinf==rangesup){
    cout<<"Errado\n";
   return 0;
  }

  //Tamanho do numero binário
  n=log(abs(rangesup))/log(2) + 1;

  //Cria numero binário de 32 bits
  bitset<32> aleatorio;

  cout<<"Range inferior: "<<rangeinf<<"\nRange Superior:  "<<rangesup<<endl;
  cout<<"Quantidade de Bits: "<<n<<endl;
  cout<<"Botao 'z' gera numero aleatorio\n";

  namedWindow("Video",WINDOW_AUTOSIZE);

  while(1){
  //Captura duas imagens
  cap >>image;
  cap>>image1;
  //Transforma em tons de cinza
  cvtColor(image, frame, CV_BGR2GRAY);
  cvtColor(image1, frame1, CV_BGR2GRAY);

  //Comandos
  key = (char) waitKey(10);
  if( key == 27 ) break; //Esc - Para programa
    if( key == 122 ){  //Z - Gera numero
      cout<<"Gerando Numero Aleatorio\n";
      posicao=0; // Zera posição do número binário
      aleatorio.reset();  // Reseta número binário
    }
  //Verifica posição
  if(posicao>=0){
     //Laço para pegar o primeiro pixel diferente entre imagens
     for(i=x;i<height;i++){
      for(j=y;j<width;j++){
	if(y>=width) y=0;
	if(x>=height) x=0;
 	if(frame.at<uchar>(i,j)!=frame1.at<uchar>(i,j)){
	  teste=frame1.at<uchar>(i,j);
	  z=1;
	  break;
	}
      }
  	if(z==1) break;
    }
     x=i+1;
     y=j+1;

     /* Coloca valor do pixel em binario
        Designa valor para posicao do numero binario   */
     aleatorio[posicao]=teste%2;
     z=0;

     //Incrementa a posicao
     posicao+=1;
    }
    //Numero binario completo, entra nesse if
    if(posicao>=n){
      /* Transforma numero binario em inteiro
	 Modula numero para o range		*/
      cout<<"Numero: "<<aleatorio.to_ulong()%(rangesup-rangeinf+1)+rangeinf<<endl;
      posicao=-1;
    }

    imshow("Video",frame);

  }
  return 0;
}

Percebemos que inicialmente alguns tipos de variáveis são criadas e posteriormente o programa iniciliza a captura de vídeo. Um detalhe importante é que a câmera deve estar totalemente tampada para que o programa funcione corretamente.

Existe um if para verificar se a captura está ocorrendo, caso contrário o programa fecha.

São retirada as dimensões das imagens precessadas e guardadas em duas variáveis, adiante é informada ao usuário sobra as dimensões e perguntado os limites inferiores e superiores na qual deva estar o número.

Essa parte é essencial para que o programa não trave. O primeiro 'IF' verifica se os limetes estão trocados, caso sim ele arruma automativamente. Já o segundo 'IF' verifica se os limites são iguais, caso seja o programa irá fechar.

Analisamos o próximo código:

 //Tamanho do numero binário
  n=log(abs(rangesup))/log(2) + 1;

Essa função nos dirá a quantidade de bits necessário para maior número desejado pelo usuário na qual é o limite superior.

Para encontrarmos essa equação foi necessário o seguinte.

\(2^n=rangesup \to nlog_{10}(2)=log_{10}(rangesup)\\ n=\frac{log_{10}(rangesup)}{log_{10}(2)}+1\)

Esse número '1' somente foi colocado pois a variável 'n' é inteiro e sempre arredonda para baixo as casas decimais, entam a utilização dessa soma é para compensar caso tenha número quebrado.

Posteriormente é criado um número binário e informado ao usuário os limites e a quantidade de Bits utilizados.

Apertando o botão 'z' o gerador de número aleatórios entragará um número entre os limites solicitados.

Agora iniciaremos no laço como demostrado no Fluxograma, primeiramente é capturado duas imagens e transformados em tons de cinzas para ajudar no processamento.

Verifica no 'if' se a posição é maior que zero, caso não seja o gerador fica esperando algum comando.

Entrando nesse 'if', imadiatamente entra em dois laços que poderá varrer toda a imagem.

Dentro do laços possui três 'if', o último serve para pegar o primeiro pixel diferente entre as imagens, e designando a variável 'z' o valor um.

O primeiro e segundo 'if' serve para zerar as variáveis 'x' e 'y' caso o laço estrapole a imagem.

Posteriormente do laços, percebemos que as variáveis 'x' e 'y' recebe o valor de 'i' e 'j' somado mais um, isso ocorre para que no próximo laço não pege o mesmo pixel diferente e sim outro.

Os quadrados brancos são considerados com pixels diferentes, percebe que no momento que j anda uma unidade e encontra um pixel, automaticamente um bit é gerado para o número aleatório.

Sucessivamente a variável 'x' e 'y' recebe repectivamente os valores das variáveis 'i' e 'j' e incrementa uma unidade. Com isso ele continua a procura de outro pixel até o momento que a quantidade de bits sejam atendidas.

Antes de sair do laço a variável binária recebe na posição determinada o valor do pixel em binário.

Finalizando o laço ocorre a incrementação da possição e a zeração da variável z.

No momento que o número estive totalmente completado irá entrar nesse 'if'

//Numero binario completo, entra nesse if

Para esse caso, a 'if' verifica se a posição é igual ao tamanho da variável desejável para o número aleatório.

O programa transforma o número binário em inteiro e posteriormente modula para os limites que o usuário informou no início do programa.

A variável 'posição' recebe o valor de -1 justamente para não ficar em um laço infinito de geração de números aleatórios.

O programa fica amostrando as imagens do vídeo justamente para o usuário fazer os ajustes necessário para que o programa trabalhe normal.

Conclusão

Nesse relatório foi notório as diferenças entre os números aleatórios e pseudoaleatórios, foi demostrando gerados dos dois lados e suas principais caracteristicas.

Os números aleátorios apresentam maior confiabilidade das informações, podendo dizer que é impossível nos dias atuais descobrir qual número será mostrado pelo simples motivo que o gerador está trabalhando em cima de ruído.

Para utilidade comum ou simples os números pseudosaleatórios são muito eficazes e aceitáveis, mas para utilizadade mais avançada que mexe com dados importantes e sigilosos é recomendado utilizar números aleatórios pois teŕa mais dificuldade de invasores.

Os dois tipos de geradores trabalham em ambientes diferentes que se complementam, os dois apresentam pontos fracos e fortes diantes de certas situações.
